## 图片链接转换和部署计划

### 任务目标
1. 修复 `src/content/spec` 目录中所有文件的图片链接
2. 确保图片能够在网站主页正确显示
3. 部署项目并验证修复效果

### 具体步骤

#### 第一阶段：文件分析
1. **扫描 spec 目录中的所有文件**
   - 列出 `src/content/spec` 目录中的所有 Markdown 文件
   - 统计文件数量（目前发现约 23 个文件）

2. **识别图片链接**
   - 搜索所有包含 `/wp-content/` 的相对路径图片链接
   - 搜索所有来源于 `hhj520.top` 及其子域的图片链接
   - 分类记录需要转换的链接类型

#### 第二阶段：批量转换
3. **处理相对路径图片链接**
   - 查找以 `/wp-content/` 开头的图片链接
   - 转换为 `https://cdnkp.hhj520.top/wp-content/...` 格式

4. **处理域名图片链接**
   - 查找来源于 `hhj520.top` 的图片链接
   - 保留路径部分并添加 CDN 前缀

5. **保留不需要转换的内容**
   - 非相对路径且不来源于 hhj520.top 的图片链接保持不变
   - 所有视频链接保持不变
   - 其他文章内容正常保留

#### 第三阶段：验证和部署
6. **本地验证**
   - 运行 `pnpm install` 安装依赖
   - 运行 `pnpm run build` 构建项目
   - 检查构建是否成功
   - 验证图片链接是否正确

7. **部署到生产环境**
   - 根据项目配置进行部署
   - 验证所有图片是否正常显示

#### 第四阶段：用户确认
8. **等待用户确认**
   - 展示修复后的链接示例
   - 提供部署结果报告
   - 等待用户确认是否清理

### 预期产出
- 所有 spec 目录中的图片链接正确转换
- 项目成功构建和部署
- 所有图片能够在网站主页正常显示

### 注意事项
- 保持原有排版格式不变
- 仔细检查每个文件的转换准确性
- 排除所有视频链接和非相关链接
