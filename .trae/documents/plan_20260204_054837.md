## 修复 spec 目录文件格式问题计划

### 任务目标
1. 扫描 `src/content/spec` 目录中的所有 Markdown 文件
2. 检查并修复格式问题
3. 部署项目并验证修复效果

### 具体步骤

#### 第一阶段：文件扫描
1. **列出所有文件**
   - 统计 `src/content/spec` 目录中的文件数量
   - 识别所有 Markdown 文件（.md 后缀）

2. **检查格式问题**
   - Frontmatter 格式检查
   - YAML 语法验证
   - 图片链接格式检查
   - 特殊字符处理

#### 第二阶段：批量修复
3. **修复 Frontmatter 问题**
   - 确保所有必要字段存在且格式正确
   - 修复标签和分类格式

4. **修复 YAML 语法错误**
   - 处理多行文本
   - 转义特殊字符
   - 确保缩进正确

5. **优化图片链接**
   - 保持 CDN 格式正确
   - 排除视频链接

#### 第三阶段：部署验证
6. **本地测试**
   - 运行 `pnpm install` 安装依赖
   - 运行 `pnpm run build` 构建项目
   - 验证无构建错误

7. **部署项目**
   - 根据平台配置进行部署

### 预期产出
- 所有 spec 目录中的文件格式正确
- 项目成功构建和部署
- 无任何错误或警告
