# 添加中文导航链接和自定义 spec 页面链接

## 问题分析

用户希望：

1. 使用中文的导航预设值，如 `LinkPreset.朋友圈`、`LinkPreset.时间线`、`LinkPreset.关于`
2. 添加两个新链接指向 spec 页面：
   - 指向 `/spec/page-1770175976.599339`，标题为"生活锁记"
   - 指向 `/spec/page-1770176032.226729`，标题为"朋友圈"

**限制**：
- `LinkPreset` 是一个枚举类型，只定义了 `Home`、`Archive` 和 `About` 三个值
- 中文预设值在枚举中不存在，不能直接使用

## 解决方案

### 1. 使用自定义链接对象代替预设值

在 `navBarConfig.links` 数组中，将中文预设值替换为自定义链接对象，保持中文标题。

### 2. 添加新的 spec 页面链接

在 `navBarConfig.links` 数组中添加两个新的链接对象，指向指定的 spec 页面。

## 实施步骤

### 1. 修改 navBarConfig 配置

```typescript
export const navBarConfig: NavBarConfig = {
	links: [
		{
			name: "朋友圈",
			url: "/",
			external: false
		},
		{
			name: "时间线",
			url: "/archive",
			external: false
		},
		{
			name: "关于",
			url: "/about",
			external: false
		},
		{
			name: "生活锁记",
			url: "/spec/page-1770175976.599339",
			external: false
		},
		{
			name: "朋友圈",
			url: "/spec/page-1770176032.226729",
			external: false
		},
		{
			name: "旧站",
			url: "https://cdnkp.hhj520.top",
			external: true
		},
	],
};
```

### 2. 验证配置

运行构建命令验证配置是否正确，确保链接能正常工作。

## 预期效果

导航栏将显示以下链接：
1. 朋友圈（指向首页）
2. 时间线（指向归档页）
3. 关于（指向关于页）
4. 生活锁记（指向 `/spec/page-1770175976.599339`）
5. 朋友圈（指向 `/spec/page-1770176032.226729`）
6. 旧站（指向外部链接）

这样用户就可以使用中文标题，并且添加自定义的 spec 页面链接。
