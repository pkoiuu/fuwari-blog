## 问题分析

评论区滑动时排版错乱的原因：

1. **容器高度不稳定** - min-height: 100px 不够，Twikoo 加载后高度变化导致布局跳动
2. **延迟加载闪烁** - Intersection Observer 触发时内容从无到有
3. **缺少加载占位符** - 无占位符，用户看到空白

## 修复方案（性能优先）

### 1. 适度增加容器最小高度
- 将 min-height 从 100px 增加到 200px（平衡体验和性能）

### 2. 轻量级加载占位符
- 使用纯 CSS 动画，不增加 JS 负担
- 简单的骨架屏效果

### 3. 优化 Intersection Observer
- 保持延迟加载（性能关键）
- 适度增大 rootMargin 到 200px（提前加载但不过早）

### 4. CSS  containment 优化
- 使用 contain: layout 防止布局抖动
- 使用 content-visibility 优化渲染

### 5. 平滑过渡
- 使用 CSS transition 替代 JS 动画
- 零 JS 开销

## 性能保障

- ✅ 保持 Intersection Observer 延迟加载（不阻塞首屏）
- ✅ 使用纯 CSS 动画（无 JS 运行时开销）
- ✅ CSS containment 防止重排重绘
- ✅ 不增加额外的网络请求
- ✅ 不增加第三方库
